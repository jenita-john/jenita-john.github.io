Git Cheat Sheet

In this cheatsheet you will find the most basic git commands for everyday use. It is intended to help with common Git issues.

1. First GitHub Commit:

You have made all the changes and ready to push your first commit. Here's what you must do first, check the status and diff before committing.

 
 git status
 git diff

status will list all the files you modified while the diff will show the changes you made. When you are happy with the changes, move on to next step and add the files.

Only add the files that needs to be committed, DO NOT git add .
This will add all the files to the commit. 


 git add [file]

For adding multiples files at once:


 git add [file1] [file2] ...

Commit the changes with a message:


 git commit -m [descriptive message]

Now it's time to push your first commit. Since this is your first time to push a commit to a feature branch, you must setup the upstream.


 git push --set-upstream origin [branch-name]

Summary:


 git status
 git diff
 git add [file]
 git commit -m [descriptive message]
 git push --set-upstream origin [branch-name]
